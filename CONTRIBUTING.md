# Contributing to Sayr

Thanks for your interest in contributing. Before you submit a pull request, please read this document.

## Contributor License Agreement

All contributors must sign the [Contributor License Agreement](./CLA.md) before any contribution can be accepted. The CLA is enforced automatically on pull requests — a bot will post a comment with instructions when you open your first PR.

To sign, post the following comment on your pull request:

```
I have read the CLA Document and I hereby sign the CLA
```

## Getting Started

See [AGENTS.md](./AGENTS.md) for the full development setup, commands, and code style guide.

## Submitting a Pull Request

1. Fork the repo and create a branch from `main`
2. Make your changes following the code style in `AGENTS.md`
3. Open a pull request against `main`
4. Sign the CLA if prompted
5. A maintainer will review your PR

## Reporting Issues

Open an issue on GitHub. For security vulnerabilities, do not open a public issue — contact us directly at [security@sayr.io](mailto:security@sayr.io).
