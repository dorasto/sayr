# Visibility Controls

One of Sayr's core features is granular visibility control. Every piece of content can be independently set to public or private.

## How It Works

When you create any of the following, you can set its visibility:

- **Tasks**: The task itself, including title and description
- **Comments**: Individual comments on tasks
- **Timeline Entries**: Activity log entries

## Public vs Private

| Visibility | Who Can See |
|------------|-------------|
| Public | Anyone visiting your organization's public page |
| Private | Only authenticated organization members |

## Use Cases

### Open Source Projects

Make your roadmap public while keeping internal discussions private:

- Public: Tasks, priorities, status updates
- Private: Budget discussions, internal comments

### Client Projects

Share progress with clients while maintaining internal notes:

- Public: Deliverable tasks, milestone updates
- Private: Implementation details, team discussions

## Setting Visibility

### On Creation

When creating a task or comment, use the visibility toggle to set public or private.

### Bulk Updates

Organization admins can bulk-update visibility settings from the task list view.
